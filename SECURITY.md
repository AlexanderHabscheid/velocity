# Security Policy

## Supported Versions

Only the latest minor release on the `0.x` line is supported for security patches.

## Reporting a Vulnerability

1. Do not open a public issue for suspected security vulnerabilities.
2. Email maintainers with reproduction details, impact, and proof-of-concept.
3. Include affected version, deployment shape, and whether auth/policy controls were enabled.

Acknowledgement target: 2 business days.
Initial triage target: 5 business days.

## Disclosure

We follow coordinated disclosure. A fix release and advisory are published after validation.
